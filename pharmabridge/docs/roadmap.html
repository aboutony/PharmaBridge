PharmaBridge Frontend Development Roadmap

Overview

This roadmap outlines the phased development of the PharmaBridge Syria frontend based on the Frontend Technical Specification v1.0. Development follows a frontend-first approach with MSW mocking, ensuring full light/dark mode, Arabic RTL primary, English secondary, and responsive design across desktop/tablet/mobile.

Phases and Subphases

Phase FE-1: Project Setup and Core Architecture

Initialize the Next.js project with approved technology stack and establish project structure.

  • FE-1.1: Initialize Next.js 14+ with App Router and TypeScript
  • FE-1.2: Configure Tailwind CSS 3+ and shadcn/ui component library
  • FE-1.3: Set up project folder structure per specification
  • FE-1.4: Install and configure all approved dependencies (Zustand, TanStack Query, React Hook Form, etc.)
  • FE-1.5: Configure ESLint, Prettier, Husky, and lint-staged
  • FE-1.6: Set up Vercel project and GitHub Actions CI/CD pipeline

Phase FE-2: Design System and Theming

Implement design tokens, light/dark mode, and theme management.

  • FE-2.1: Define CSS custom properties for design tokens (colors, surfaces, text)
  • FE-2.2: Configure Tailwind to consume design tokens
  • FE-2.3: Implement next-themes for system-aware light/dark switching
  • FE-2.4: Create ThemeToggle component
  • FE-2.5: Build base shadcn/ui components with dark mode support
  • FE-2.6: Ensure all components pass light/dark mode QA

Phase FE-3: Internationalization and RTL/LTR

Set up bilingual support with Arabic RTL primary.

  • FE-3.1: Configure next-intl middleware for locale routing
  • FE-3.2: Create translation files (ar.json and en.json) with all strings
  • FE-3.3: Implement RTL/LTR CSS rules and logical properties
  • FE-3.4: Build LanguageToggle component
  • FE-3.5: Test all components in both Arabic RTL and English LTR
  • FE-3.6: Implement directional icons and RTL-specific adjustments

Phase FE-4: Responsive Design System

Establish breakpoint definitions and layout patterns.

  • FE-4.1: Configure Tailwind breakpoints (mobile: 320-767px, tablet: 768-1279px, desktop: 1280px+)
  • FE-4.2: Build layout templates (sidebar, auth, public)
  • FE-4.3: Implement responsive navigation (bottom nav on mobile, sidebar on tablet/desktop)
  • FE-4.4: Create ResponsiveTable component for data tables
  • FE-4.5: Ensure touch targets and spacing meet WCAG guidelines
  • FE-4.6: Test all breakpoints and orientations

Phase FE-5: MSW and Mock Data Setup

Implement frontend-first mocking with MSW.

  • FE-5.1: Configure MSW for browser and server environments
  • FE-5.2: Create mock data fixtures for all modules
  • FE-5.3: Build MSW handlers for all API endpoints
  • FE-5.4: Integrate MSW with TanStack Query hooks
  • FE-5.5: Set up environment-specific MSW activation (dev/demo only)

Phase FE-6: Core Layouts and Navigation

Build shared layouts and navigation components.

  • FE-6.1: Create root layout with theme and locale providers
  • FE-6.2: Build navigation components (Sidebar, Header, BottomNav)
  • FE-6.3: Implement portal route groups ([locale]/(pharmacy), etc.)
  • FE-6.4: Create layout templates for each portal
  • FE-6.5: Add breadcrumb and back navigation
  • FE-6.6: Test navigation across all breakpoints and themes

Phase FE-7: Authentication Module

Implement login, registration, and user management.

  • FE-7.1: Build auth forms (login, register, reset password)
  • FE-7.2: Integrate React Hook Form with Zod validation
  • FE-7.3: Create auth layout and routing
  • FE-7.4: Implement MSW auth handlers
  • FE-7.5: Add auth state management with Zustand
  • FE-7.6: Test auth flows in both languages and themes

Phase FE-8: PIMS Module - Pharmacy Inventory Management

Develop core pharmacy inventory functionality.

  • FE-8.1: Build inventory dashboard with key metrics
  • FE-8.2: Implement drug catalog with barcode scanning
  • FE-8.3: Create stock level tracking and alerts
  • FE-8.4: Add expiry date management and notifications
  • FE-8.5: Build inventory CRUD operations
  • FE-8.6: Ensure offline functionality with IndexedDB

Phase FE-9: DistributorLink Module - B2B Connectivity

Implement distributor search and ordering.

  • FE-9.1: Build distributor search interface
  • FE-9.2: Create live inventory visibility
  • FE-9.3: Implement order placement and tracking
  • FE-9.4: Add order history and management
  • FE-9.5: Build credit and payment tracking
  • FE-9.6: Test B2B flows across all breakpoints

Phase FE-10: PharmaMarket Module - B2C Marketplace

Develop patient-facing marketplace features.

  • FE-10.1: Build drug availability search
  • FE-10.2: Create pharmacy storefronts
  • FE-10.3: Implement order placement and pickup/delivery options
  • FE-10.4: Add prescription upload functionality
  • FE-10.5: Build patient dashboard and order history
  • FE-10.6: Integrate with pharmacy inventory

Phase FE-11: BridgeIntel Module - Analytics & BI

Add business intelligence dashboards.

  • FE-11.1: Build pharmacy-level analytics (sales, turnover)
  • FE-11.2: Create distributor analytics
  • FE-11.3: Implement charts with Recharts
  • FE-11.4: Add export functionality
  • FE-11.5: Ensure responsive chart rendering
  • FE-11.6: Test analytics in dark mode and RTL

Phase FE-12: Offline Mode and PIMS Integration

Enhance offline capabilities.

  • FE-12.1: Implement IndexedDB layer with idb
  • FE-12.2: Build sync queue and conflict resolution
  • FE-12.3: Add connectivity detection
  • FE-12.4: Create offline status indicators
  • FE-12.5: Test offline flows comprehensively

Phase FE-13: Testing, Storybook, and QA

Establish testing and component documentation.

  • FE-13.1: Set up Vitest and React Testing Library
  • FE-13.2: Configure Storybook with theme and RTL variants
  • FE-13.3: Create stories for all components
  • FE-13.4: Implement Playwright E2E tests
  • FE-13.5: Run pre-delivery checklist on all features
  • FE-13.6: Performance testing and optimization

Phase FE-14: Environment Configuration and Deployment

Set up demo and production environments.

  • FE-14.1: Configure environment variables
  • FE-14.2: Set up Vercel projects (demo and production)
  • FE-14.3: Implement demo banner and seeded data
  • FE-14.4: Configure GitHub Actions for automated deployment
  • FE-14.5: Set up preview deployments for branches
  • FE-14.6: Test environment switching

Phase FE-15: Accessibility and Performance Optimization

Final polish and compliance.

  • FE-15.1: Implement WCAG 2.1 AA accessibility requirements
  • FE-15.2: Optimize bundle size (<200KB gzipped)
  • FE-15.3: Ensure LCP <2.5s, CLS <0.1 on mobile
  • FE-15.4: Add ARIA labels and keyboard navigation
  • FE-15.5: Final RTL/LTR and light/dark QA
  • FE-15.6: User acceptance testing and sign-off
